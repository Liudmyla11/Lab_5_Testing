# Лабораторна робота №5: Модульне та Інтеграційне Тестування (JavaScript/Jest)

## Структура проекту

Цей репозиторій містить дві основні частини відповідно до інструкції:

1.  **`unit-integration-tests/`**: Проект-приклад, наданий у методичних вказівках (тестування `math`, `auth`, `currency` та `api`).
2.  **`lab5-my-project/`**: Власний проект для виконання самостійного завдання.

### Самостійний проект: "Утиліти для замовлень та користувача"

Проект містить 3 модульних та 2 інтеграційних тести:

| Тип тесту | Функція | Призначення |
| :--- | :--- | :--- |
| **Модульний (Unit)** | `isValidEmail` | Валідація email. |
| **Модульний (Unit)** | `formatUsername` | Форматування імені. |
| **Модульний (Unit)** | `calculateDiscount` | Розрахунок знижки. |
| **Інтеграційний (Internal)** | `placeOrder` | Перевірка email та розрахунок знижки (модуль + модуль). |
| **Інтеграційний (API)** | `fetchAndValidateUser` | Отримання даних з API та їх валідація (модуль + зовнішня залежність). |
